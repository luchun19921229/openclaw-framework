## Core Components

### Gateway

- Node.js process managing sessions and routing
- Handles message I/O across channels (Telegram, Discord, WeChat)
- Manages agent lifecycle (spawn, steer, kill)
- Cron scheduler for timed tasks

### Agent Runtime

Each agent instance:
- Reads workspace files at session start (SOUL.md, USER.md, AGENTS.md, MEMORY.md)
- Gets tool access via runtime policies
- Spawns sub-agents for complex tasks
- Writes to daily memory logs

### Workspace

File-based, no database required:
- **Root files** — Agent identity and configuration
- **memory/** — Daily logs and long-term memory
- **skills/** — Installed capabilities
- **workspace/** — Active projects and data

## Design Principles

1. **File over database** — Everything is a file, easy to read, edit, backup
2. **Agent over chatbot** — Proactive, persistent, opinionated
3. **Local-first** — Ollama/MLX before cloud APIs
4. **Skill over monolith** — Composable capabilities, not bloated code
5. **Memory over context window** — Structured memory beats prompt stuffing
